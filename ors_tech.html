OpenRegSpecs for Tool Developers

Introduction

OpenRegSpecs is a definition of file formats and associated diagrams allowing regulatory experts to describe and collaborate on regulatory artefacts

It is useful to build tools on top of this to use these artefacts, such as:

  • Text editors for the file formats with auto-completion
  • Visual and textual navigation of the artefacts
  • Translating from the artefacts into other formats such as SQL schemas
  • Translating from existing schemas/BPMN workflow definitions/Test data/requirements into the OpenRegSpecsFormat
  • Tests
To this end we have a UML model which can describe the artefacts such as data model and workflow, this UML model can store all the structured information represented in the text files. The UML class diagrams represents an information model only, they have attributes and relationships, but no methods/functions.

As part of Eclipse Free BIRD tools we have code that converts to/from the text format into an in-memory object tree using the classes defined in the UML format.

Having this in-memory representation of the files, allows any kind of manipulation/navigation/query/transformation/visualisation of the artefacts

As part of Eclipse Free BIRD tools we have an example of this which uses the Eclipse Modelling Frameworks and its Sirius visualisation component to create diagrams.

OpenRegSpecs aims to work with other datamodels and regulatory projects, not just BIRD.

Specifically for BIRD we have some extra text formats to represent artefacts specific to BIRD (such as SubDomains).

Also to support BIRD we have Python programs to translate from published BIRD Artefacts into the Python classes representing the classes of the UML diagram. This provides a full information model of all BIRD artefacts including logical transformation rules and tests, allowing for an in-memory representation to use in tool support such as validation of test data against data models.

Object Trees of these Python classes can be serialised into a common Ecore XMI format if desired, From this Ecore format the Python objects can be recreated, or Java Objects can be created also (allowing integration with the example visual tools in Eclipse Free BIRD Tools)

The Python classes have a very clean and simple interface which is easy to use, and enforces type safety.
