PyBIRD AI Workflows Overview

Introduction

PyBIRD AI provides three specialized workflows for processing Banks' Integrated Reporting Dictionary (BIRD) data. Each workflow is designed for specific regulatory reporting requirements and data processing scenarios.

Available Workflows

1. Main Workflow (FINREP/DPM)

Purpose: Traditional FINREP (Financial Reporting) data processing using the DPM model

Best for:

  • Standard FINREP regulatory reporting
  • Processing SQLDev EIL/ELDM files
  • Traditional banking regulatory submissions

Access: Navigate to Workflows > Main Workflow or visit /pybirdai/workflow/

2. ANCRDT Workflow (AnaCredit)

Purpose: Specialized processing for AnaCredit (Analytical Credit Datasets) reporting

Best for:

  • Credit reporting requirements
  • ECB AnaCredit submissions
  • Granular loan-level data processing

Access: Navigate to Workflows > ANCRDT Workflow or visit /pybirdai/ancrdt-workflow/

3. DPM Workflow

Purpose: DPM-specific data processing and output layer creation

Best for:

  • Creating DPM output structures
  • Non-standard DPM reporting requirements

Access: Through the main workflow dashboard with DPM-specific routes

Workflow Stages Overview

Main Workflow - 4 Tasks

  1. Task 1: Data Import & LDM Processing
  2. Task 2: Semantic Integration & Filters
  3. Task 3: Joins Metadata & Transformations
  4. Task 4: Execution & Testing

ANCRDT Workflow - 5 Steps

  1. Step 0: Fetch Metadata CSV
  2. Step 1: Import Metadata
  3. Step 2: Generate Joins Metadata
  4. Step 3: Generate Execution Code
  5. Step 4: Execute Tables

DPM Workflow - 3 Steps

  1. Step 1: Prepare DPM Data
  2. Step 2: Import DPM Data
  3. Step 3: Create Output Layers

Key Features Across Workflows

Metadata Editing

All workflows support editing metadata relationships:

  • Cube Links: Define relationships between data cubes
  • Cube Structure Item Links: Link variables between cube structures
  • Member Links: Map specific dimension member values

See: Edit Links Features Guide

Code Review and Editing

ANCRDT and Main workflows include code review capabilities:

  • View generated Python code
  • Edit execution logic in browser-based editor
  • Deploy code to production environment
  • Track sync status and manual modifications

See: Code Editing and Publishing Guide

Output Layer Mapping

A specialized 7-step workflow for creating mappings and output structures:

  • Select tables and ordinates
  • Define variable breakdowns
  • Create mappings
  • Generate cube structures

See: Output Layer Mapping Workflow Guide

Choosing the Right Workflow

Use Main Workflow when:

  • Processing standard FINREP requirements
  • Working with SQLDev EIL/ELDM exports
  • Following traditional DPM reporting paths
  • Need comprehensive 4-task pipeline

Use ANCRDT Workflow when:

  • Processing AnaCredit data
  • Working with ECB CSV exports
  • Need granular loan-level reporting
  • Require table-based execution with dimension filters
  • Need code deployment management

Use DPM Workflow when:

  • Creating custom DPM output structures
  • Need specialized DPM processing
  • Building non-standard reporting layers

Getting Started

  1. Determine your reporting requirement - FINREP, AnaCredit, or custom DPM
  2. Select the appropriate workflow from the navigation menu
  3. Follow the step-by-step process in each workflow
  4. Review generated structures at each stage
  5. Execute and validate your data transformations
