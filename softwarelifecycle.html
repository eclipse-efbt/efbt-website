Task Focussed Development, Debug , Testing , and Change Management

Eclipse Free BIRD Tools is designed to provide maximum tool support for the common tasks in development, debug , testing , and change management of programs written with VTL and SMCubes.

Development

The task-focussed attribute lineage model gives us a fine grained view of dependencies between artifacts such as report cells and columns.

It also allows us to exclude the hundreds or thousands of elements (report cells, calculation, intermediate columns, input layer columns) which are not related to our task.

This allows us to answer questions such as
  • Which report cells will be affected if I change this line of VTL code?
  • Which report cells will be affected if I delete or change the name of this input column?

Debugging

The task-focussed data lineage model gives us a fine grained view of the numbers and transactions used to calculate a value in a report cell.

It also allows us to exclude the thousands or millions of elements (report cells, calculations, intermediate results, input layer column values) which are not relevant to our task.

The data lineage model follows the familiar concepts of basic spreadsheets with formula.

This allows us to answer questions such as
  • How is the value 10 in report cell X calculated?
  • Which individual transactions and which numbers from those transactions went into making report cell X populated as 10?

Testing

. Our testing model allow us to answer questions such as
  • Which business concepts and scenarios are being tested by this test?
  • Which sections of regulatory text are being checked in this test?
  • What percentage of the code base are my tests covering?

Change management

Our fine grained task-focussed attribute lineage model of dependencies can be compared between versions of a program to answer questions such as
  • Which report cells have zero change in their calculation between version x and version y of a VTL program?
  • Did the usage of a particular column in the input layer change between 2 versions of a VTL/SMCubes program?
