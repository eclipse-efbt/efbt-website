Task Focussed

Save time where you spend the most time.

Tool support for your most common tasks.

The goal of Eclipse Free BIRD Tools is to be a task oriented environment , providing the maximum tool support possible in completing the common time consuming tasks to be done while developing, changing , or using a regulatory calculation engine written using VTL and SMCubes, such as BIRD.

The goal is to speed up the tasks, and this is the overriding principle in the entire design of Eclipse Free BIRD Tools.

Example tasks are:

  • Debugging, e.g. why is report cell X equal to 10, when I think it should be 20?
  • Comprehending the logic, e.g. how is report cell X calculated?
  • Understanding calculated results, e.g. which individual transactions and which numbers from those transactions went into making report cell X populated as 10?
  • Managing change, e.g. which report cells will be affected if I change this line of code, or delete this input column.
  • Mapping data from financial systems, e.g. which items of the data model do I need to populate to produce just this single report?
  • Testing, e.g. which cubes and which individual columns of the input layer do I need to provide test data for to test that report cell X is calculated correctly.
  • Quality control, e.g. exactly which code was changed when bug X was previously fixed, who changed it, when, who reviewed it and which test tested the change?
We achieve this goal with the following 3 features:

Task Oriented Attribute Lineage model

Divining just the small set of attributes you need for a task

Spreadsheet Data Lineage

Datalineage described in a way familiar to Excel users

SMCubes XML

Enabling fine grained source control for BIRD artefacts
