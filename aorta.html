AORTA: An Open Regulatory Testing Architecture

AORTA

AORTA provides a model from Semantic transformations which gives a business user friendly breakdown of regulations as a workflow of tasks.

AORTA also provides a simple model for storing regulatory text, and tasks in the workflow can therefore link to regulatory text.

AORTA also provides a model for test definitions which reference the tasks so that test coverage can be measured.

AORTA also provides a model for tests, test input data, and expected results, which links to the SMCubes model, so it can represent input data and report data in the BIRD format.

AORTA uses a custom model for a workflow of tasks, however we are currently looking at replacing this with a cutdown version of BPMN . We do not consider it a good idea to use all of BPMN as this would provide obstacles to providing good linkage to the Logical Data Model and technical transformation rules.
