Overview

Eclipse Free BIRD Tools aims to provide a free set of tools to aid the development and understanding of the BIRD artefacts.

It also aims to enable and fast track the development of commercial tools through the sharing of re-usable models and software components.

The sharable components come in 4 parts, and each one builds on the previous. These are:

Universal and Unified formats

To simplify the collaboration on BIRD artefacts we use universal commonly understood formats such as CSV, structured text (markdown), and json so that we avoid the need for learning a new format before participants can contribute.

For example test data is in CSV, business requirements in markdown text, and lineage information is in JSON.

Using this approach we avoid the need for learning or purchasing custom tools for contribution to the BIRD, although universal free tools like Excel exist for editing csv files.

To allow full navigation and automatic validation across all information, we also have a unified format which stores all information (data models, transformation, tests, requirements) in one unified model described using the UML standard (Unified Modeling Language)

We can always convert from the universal formats to the unified format and vice-versa. The Unified Model is split into "BIRD Data Models" for data structures and also "Business Requirements and Testing Models" as described below.

SMCube Data Models

Eclipse Free BIRD Tools provides a set of UML class diagrams to describe the SMCubes Data Model

These follow the SMCubes UML model used by the ECB to describe the BIRD Artefacts.

These models are provided as documented diagrams at Model Documentation and are completely technology independent.

Note that the class diagrams have only attributes and relationships , and no operations, which keeps them simple.

Calculation Lineage

Eclipse Free BIRD Tools provides a set of UML class diagrams named CoCaLiMo for describing the lineage and workflow of regulatory calculations.

These models also track and record the lineage of regulatory calulations to regulations and business requirments, and flesh out business requirements with specific scenarios that should be covered. These models are provided as documented diagrams at Model Documentation and are completely technology independent.

Note that the class diagrams have only attributes and relationships , and no operations, which keeps them simple.

Calculation Lineage Extensions for BIRD Data Models

Eclipse Free BIRD Tools provides a set of UML class diagrams which allows for test data to be recorded according to SMCubes data structures and linked to the Scenarios described in the calculation lineage models

These models are provided as documented diagrams at Model Documentation and are completely technology independent.

Note that the class diagrams have only attributes and relationships , and no operations, which keeps them simple.

A set of reusable software components

Using the mature and well supported open source Eclipse Modelling Framework, we provide a set of software components for visualizing, navigating, creating, editing, persisting, querying, transforming, migrating, refactoring, and forward engineering models and instances of the models.

The project also provides translators from the universal formats to the the unified model by using the Eclipse Modelling Framework to represent instances of the unified format .

These components are released with a commercially friendly open source license, and can be used in commercial software.

These components are available on the EFBT Github Repository along with a wiki giving technical advice of how to build and deploy them. They are also combined into a usable application.

The Eclipse Modelling Framework itself has many companies offering free and commercial support, and it is used in for industry by companies such as Bosch, Seimens , Thales, Nasa, and also in a number of banks.
